Technical SIEM Manifest: Agentic Node and Edge Master Registry
1. Document Overview and Architectural Standards
This registry defines the authoritative Directed Acyclic Graph (DAG) for the Agentic SIEM, establishing a high-fidelity blueprint for cyber-physical security orchestration. The architecture is predicated on the modular integration of specialized agents, skills, and operations, ensuring a telemetry-to-remediation pipeline that is both explainable and resilient against cyclic logic errors.
1.1 Architectural Standards
All system definitions must adhere to the following schema for ingestion by the graph_constructor.py utility.
Node Schema: Node_ID, Node_Type, Parent_Node_ID, Depth_Level, Description
Edge Schema: Source_ID, Target_ID, Edge_Type, Description
1.2 Integration Constraints
Absolute Grounding: All Level 3 Operations must map to a Level 2 Skill and Level 1 Agent.
Atomic Integration: Updates are performed in "Simulation Mode" using a main_graph.copy() operation to prevent production outages.
DAG Validation: The networkx library enforces the is_directed_acyclic_graph property.
Referential Integrity: The Graph Constructor validates that every Parent_Node_ID exists within the Node_ID namespace.

4. Operational Appendix: Integration and Validation Commands
To maintain the architectural integrity of the /agents and /onboarding directories, the graph_constructor.py utility in /graph_logic must execute the following validation sequence before any commit to the production SIEM.
4.1 Atomic Conflict Simulation
Before integrating any new CSV from the /onboarding directory, a conflict simulation is initiated. This prevents mutation of the live state until the new path is proven stable.
# Staging simulation
test_graph = main_graph.copy()
test_graph.add_nodes_from(onboarding_data)
test_graph.add_edges_from(onboarding_edges)
4.2 Directed Acyclic Graph (DAG) Verification
To prevent infinite logic loops, the Integrity_Validation_Skill executes a cycle detection check.
import networkx as nx
if not nx.is_directed_acyclic_graph(test_graph):
    raise Exception("Integration Rejected: Cyclic dependency detected.")
4.3 Referential Integrity Check
The system must ensure that every Level 2 and Level 3 node is correctly parented to a pre-existing or simultaneously onboarded Level 1 Agent.
for node, data in test_graph.nodes(data=True):
    parent = data.get('Parent_Node_ID')
    if parent != 'None' and parent not in test_graph.nodes:
        raise ValueError(f"Referential Integrity Error: Parent {parent} not found for node {node}.")
4.4 Explainable AI (XAI) Snapshots
For compliance and debugging, the system exports periodic snapshots of the global state. These facilitate visual auditing of the system's "Chain of Thought" evolution.
Export Format: .graphml
Audit Directory: /graph_logic/snapshots/
