Technical Manifest: Unified Agentic SIEM and Cyber-Physical Registry
1. Executive Architectural Paradigm: The Graph-as-Code Framework
Security operations have reached a point of diminishing returns with traditional, linear workflows and brittle playbooks. This manifest details a strategic shift to a Directed Acyclic Graph (DAG) architecture, where security logic is managed as code. By implementing "Agentic" logic, we transform static detection and response into an autonomous lifecycle. In this framework, security tasks are not just sequences but dynamic nodes in a graph that allows for complex, multi-path orchestration from initial telemetry ingestion to remediation and long-term memory.
The system is defined by a rigorous three-tier structural hierarchy to ensure modularity and system integrity:
Level 1: Agents: High-level orchestrators responsible for specific security domains.
Level 2: Skills: Domain-specific capabilities possessed by an agent (e.g., log normalization or playbook synthesis).
Level 3: Operations: Atomic, executable tasks that serve as the leaf nodes of the graph (e.g., executing KQL queries or issuing EDR commands).
The stability of this system is mathematically guaranteed by the DAG property. By utilizing the networkx library to enforce this constraint, the architecture prevents cyclic dependencies—circular loops that could lead to infinite execution cycles or system deadlocks. This mathematical rigor ensures predictable data flow and enables atomic updates to the security posture. This architectural theory underpins the specialized digital and physical defense agents defined in the registry below.
2. Core Digital Defense: The Primary Agent Cluster
The digital security lifecycle is managed by a cluster of five primary agents. These agents coordinate the transformation of raw telemetry into actionable intelligence, ensuring deep investigation and safe remediation while maintaining a continuous learning feedback loop.
Primary Agent Profiles
Triage Enrichment Agent: Orchestrates the transition from raw log streams to high-fidelity alerts. It performs OCSF schema normalization and dynamic intelligence stitching, utilizing heuristic deduplication and graph-based asset clustering to escalate only validated, enriched incidents to the investigator layer.
Autonomous Investigator Agent: Serves as the main analyst orchestrator, utilizing "Chain-of-Thought" planning to formulate investigative hypotheses. It performs cross-telemetry pivoting across SIEM, EDR, and Cloud environments, reconstructing process trees and artifacts to isolate the "Patient Zero" root cause.
Remediation Response Agent: Focuses on real-time mitigation through the synthesis of dynamic SOAR playbooks. It enforces critical "Pre-Flight" simulations and safety guardrails, checking asset exclusion lists and change windows before executing infrastructure containment commands like host isolation or session revocation.
Knowledge Memory Agent: Acts as the system’s persistent intelligence store. It manages hierarchical memory via RAG (Retrieval-Augmented Generation), vectorizing architecture wikis and internal policies. It maintains a feedback loop that tunes agent prompt weights and embedding distances based on human analyst overrides.
Proactive Threat Hunting Agent: Identifies stealthy risks through baseline anomaly detection, adversarial emulation, and long-tail entropy analysis. It proactively searches for indicators of lateral movement and command-and-control (C2) activity that bypass signature-based detections.

The reach of this architecture extends beyond the digital network, integrating physical and IoT domains into a unified security posture.
3. Cyber-Physical Convergence: Physical SOC and IoT Agents
A comprehensive security strategy requires a unified Cyber-Physical Security Operations Center (PSOC). By treating physical sensors—such as badge readers, motion detectors, and IoT hardware—as security endpoints, we can detect threats that cross the physical-digital divide, such as unauthorized facility access followed immediately by a sensitive network login.
Specialized Physical and IoT Roles
Physical Threat Intelligence Agent: Correlates physical environmental telemetry with digital signals. It evaluates geospatial consistency by comparing the timestamp and location of physical badge swipes with VPN or system login events to identify compromised credentials or "impossible travel."
Physical Incident Response Agent: Manages automated physical countermeasures, such as locking doors or triggering silent alarms. It includes "Physical Guardrail" validation, which ensures automated lockdowns never violate fire code egress compliance or life-safety protocols.
Device Lifecycle Agent: Manages "Edge Asset Contextualization" for IoT hardware. It establishes behavioral baselines for fixed-function devices (e.g., cameras only communicating with an NVR) and assigns "Trustworthiness" scores based on firmware versions and MAC address registry data.

4. System Governance: Integrity, Oversight, and Validation Agents
To prevent "model drift" and maintain the structural integrity of the autonomous system, an oversight layer is required. This layer ensures that the underlying DAG remains stable and that the intelligence used by the system is based on verified data.
Oversight Agents
Data Integrity & Verification Agent: Responsible for auditing "ground truth." It validates the accuracy of ingested logs and filters out low-confidence analyst overrides, protecting the training signal from corruption or bad-actor feedback injection.
Graph Integration & Monitoring Agent: Acts as the architectural "Gatekeeper." It validates every new node or edge definition added to the system, simulating its impact to ensure the production environment remains a valid DAG.
Graph Monitoring Operations
The Graph Integration & Monitoring Agent utilizes Python-based logic to perform the following:
Simulate_DAG_Path_Integrity: Utilizes networkx to run the function nx.is_directed_acyclic_graph(G) on proposed updates in a staging environment to detect potential circular logic.
Reject_Cyclic_Dependencies: Automatically blocks any integration attempt that fails the DAG verification and issues high-priority alerts to the engineering team.
Audit_Integration_History: Maintains a persistent Python-based log (JSON or structured text) of all successful and failed file integrations for compliance and forensic debugging.
This governance layer ensures that all master data structures are verified before being merged into the production registry.

Onboarding Lifecycle Workflow
To maintain this registry, the system utilizes a 4-step onboarding lifecycle:
Draft in Staging: New agent, skill, or operation CSV rows are placed in an /onboarding directory.
Conflict Simulation: The system creates a temporary graph copy using main_graph.copy().
DAG Verification: The Graph Monitoring Agent runs nx.is_directed_acyclic_graph(G) to confirm no circular logic is introduced.
Atomic Merge: Upon successful validation, the new rows are merged into the master manifests, and the production graph is updated.
This integrated manifest ensures a self-healing and self-optimizing posture, allowing the security architecture to evolve as a stable Graph-as-Code entity.
